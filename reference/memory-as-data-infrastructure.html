Reference 0013

Memory as Data Infrastructure

Use this sheet to decide what to write to an agent's long-term memory, classify each record, attach governance metadata, and retrieve the right record under a token budget.

The Rule

Every durable record carries provenance, owner, timestamp, confidence, and expiry. Retrieval ranks by authority and recency, not similarity alone, and filters by permission first. A store missing those is a transcript, not memory.

Three Stores

Episodic

What happened

Past interactions, preferences, history. Scoped to a user or account. Who wanted what, when.

Procedural

What worked

Tool sequences, parameters, latency, success rates. Reusable skill knowledge. How to act well next time.

Scratchpad

Right now

Current task state and intermediate results. Discarded at task end. Filter before promoting any of it to durable memory.

Three Different Updates

  • Memory update — write a record. Cheap, instant, reversible, scoped to who retrieves it.
  • Policy / threshold update — change a rule, prompt, trigger, or evaluator. Applies to every run; needs review and rollout.
  • Model-weight update — fine-tune or retrain. Slow, expensive, hard to reverse. Not what "learning from feedback" via memory does.

Reflexion and in-context learning improve behavior by writing memory and re-injecting it — never by touching weights. Match each change to the system that should own it.

Record Anatomy

  • Provenance — where it came from (this run, tool logs, human, external source).
  • Owner / scope — whose record it is; who may retrieve it.
  • Timestamp — when written, for recency ranking.
  • Confidence — how trusted; low-confidence entries should expire fast.
  • Expiry / retention — when it goes stale or must be purged (privacy, policy).

Retrieval Order

  • 1. Hard filters — drop records the caller lacks permission for, past expiry, or out of scope (no cross-account leakage).
  • 2. Rank survivors — relevance × recency × authority (policy/approval outrank anecdotes), small confidence weight. The Generative Agents stream scores recency + importance + relevance.
  • 3. Inject the top 3–5 — respect the token budget; a long window does not read everything uniformly (Lost in the Middle).
  • 4. On conflict — higher authority, then more recent; escalate if a high-stakes conflict is unresolved.

Anti-Patterns

  • Auto-writing every user message verbatim — poisoned memory and a prompt-injection vector.
  • Storing a system-wide rule as one user's episodic record, where it can drift or over-apply.
  • Reaching for fine-tuning when a memory write would capture the change instantly.
  • Retrieval by similarity alone, surfacing stale, low-confidence, or out-of-scope records.
  • No expiry, so the store grows stale and risks privacy/retention violations.
  • Promoting the scratchpad to durable memory unfiltered, filling the store with noise.

Review Question

Before storing a "learning," ask: is this a memory write, a policy change, or a weight update — and once stored, can retrieval rank it by authority and recency, or only by how similar it reads?
