Reference 0001

Agentic Control Loop

Use this sheet to decide whether a system is merely predictive, generative, or genuinely agentic.

The Test

A system is agentic when its architecture is organized around closed-loop action in an environment: it perceives state, reasons against a goal or rule, acts through a tool or effector, observes the result, and continues or stops. This framing follows the local architecture notes and the classic agent definition from Artificial Intelligence: A Modern Approach.

PerceiveRead raw environment signals. InterpretTurn inputs into observations. ReasonApply goals, rules, constraints. ActCall tools or effectors. ObserveCheck what changed. UpdateChange state, logs, memory, or stop.

Classification

Predictive

Input to estimate

Maps data to a label, score, forecast, rank, or decision. It may inform an agent, but does not itself manage an action loop.

Generative

Prompt to artifact

Produces text, code, images, audio, plans, or other artifacts. It becomes part of an agent only when connected to controlled action and feedback.

Agentic

Goal to action loop

Uses models, rules, tools, state, and monitoring to act in an environment until success, failure, approval, or a stop condition.

Design Questions

  • What is the environment, and which parts are observable, hidden, stale, delayed, noisy, or adversarial?
  • What actions can change that environment?
  • Which actions require approval because they are irreversible, expensive, unsafe, or externally visible?
  • What feedback arrives after each action?
  • What state changes after feedback: task state, environment state, logs, memory, policy, or nothing?
  • What stop condition prevents open-ended tool use?

First Failure Mode

Do not call a prompt plus tools an architecture. Tool access without explicit state, permissions, feedback, auditability, and stop conditions is an uncontrolled action surface.
