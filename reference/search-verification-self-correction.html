Reference 0012

Search, Verification, and Self-Correction

Use this sheet to decide when a single reasoning pass is not enough, which extra-pass pattern fits, and whether the extra pass actually earns its cost.

The Rule

An extra reasoning pass adds reliability only when it brings something the first pass lacked: a different path, a different strategy, or independent evidence. A re-read of the same context by the same model mostly adds cost and false confidence.

Three Patterns

Explore

LATS

Tree search proposes several next actions, scores by feedback or rollout, commits the best one. For tasks where early mistakes are hard to undo. Cost: many calls per real action.

Generate

Self-Discover

Compose a task-specific reasoning structure first, then execute linearly by it. For novel problems with no standard shape. Risk: less predictable, harder to debug.

Verify

Second-pass verifier

Independent audit of planner output, pass/fail with explanation, before the action runs. Supports graded verdicts. Risk: false negatives if not truly independent.

Default

Single pass

For low-stakes, reversible work, one pass is the right answer. Any extra pass is pure cost unless the stakes or novelty justify it.

LATS vs. Self-Discover

  • LATS is a sequence explorer: evaluates multiple paths at each step, mid-task.
  • Self-Discover is a strategy generator: front-loads a thinking structure, then executes linearly.

Independence Test For A Verifier

  • Separate instance, model, rule system, formal checker, or human — not the generator in the same session.
  • Holds evidence the generator did not: a live re-query, an external tool, an authoritative source.
  • Returns a verdict the controller treats as the planner's next observation.
  • Graded: fatal blocks execution; minor notifies a human; pass releases the action.

Anti-Patterns

  • Treating any second pass as reliability, regardless of what it adds.
  • A self-review that re-reads the same context with the same model and no new evidence.
  • Multi-iteration reflection loops on tasks with no external signal to learn from.
  • Running LATS or Self-Discover on low-stakes, reversible tasks a single pass handles.
  • A verifier that shares the planner's model and prompt, so it inherits the same blind spots.

Review Question

Before adding a pass, ask: does this pass bring a new path, a new strategy, or independent evidence — or is it the same mind grading its own work?
