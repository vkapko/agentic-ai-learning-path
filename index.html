Learning Path

Agentic AI System Design

A sequence of focused lessons on designing autonomous AI systems — control loops, agent types, tools, memory, coordination, and the trade-offs behind each choice. Work through them in order or jump to what you need.

Lessons

  1. Lesson 0001

    Agentic AI Is a Control Loop

    Classify a system as predictive, generative, or agentic, then map one agentic workflow into perceive, interpret, reason, act, observe, and update steps.

  2. Lesson 0002

    Pick the Simplest Agent That Fits

    Classify a system as simple reflex, model-based reflex, goal-based, or utility-based, justify the smallest type that fits, and name the guardrails each step up the autonomy ladder demands.

  3. Lesson 0003

    Implement the Loop in Python

    Choose when to use the Claude Python SDK directly and when to use LangGraph, then identify where the agentic control loop lives in each example.

  4. Lesson 0004

    Decide When More Agents Are Worth It

    Choose between single-agent, centralized multi-agent, decentralized, and hybrid designs, then name the new failure modes introduced by coordination.

  5. Lesson 0005

    Choose the Smallest Reasoning Pattern

    Select one-shot generation, decomposition, ReAct, reflection, or episode memory based on the pressure in the workflow.

  6. Lesson 0006

    Design Tools As Control Boundaries

    Turn a vague agent capability into a tool contract with purpose, schema, permissions, side effects, error states, and observability.

  7. Lesson 0007

    Draw the Tool Orchestration Plan

    Turn a workflow into a small execution plan that marks dependencies, parallel calls, approval gates, failure paths, and the final synthesis step.

  8. Lesson 0008

    Design the Runtime Safety Net

    Define what an agent run must log, what signals trigger intervention, and how the system recovers from partial failure without hiding uncertainty.

  9. Lesson 0009

    Design the Extraction Contract

    Turn a vague extraction request into a schema, validation rule, and retry policy that reduces syntax errors without pretending semantics are solved.

  10. Lesson 0010

    Preserve the Case File

    Design a context packet that keeps exact facts, escalation reasons, and claim-source links intact as an agent workflow gets longer.

  11. Lesson 0011

    Divide the Work, Keep One Truth

    Pick a coordination shape for a multi-agent task, assign worker roles and tool boundaries, and design the shared-state contract that keeps every agent reading the same truth.

  12. Lesson 0012

    When One Pass Is Not Enough

    Choose between exploring action paths (LATS), generating a reasoning strategy first (Self-Discover), and an independent verifier — and judge what actually makes a second pass add reliability instead of just cost.

  13. Lesson 0013

    Memory Is Data Infrastructure

    After an agent finishes a task, decide what to write to long-term memory, classify each record, attach governable metadata, and write a retrieval rule that ranks by authority and recency — not similarity alone.

  14. Lesson 0014

    Human-in-the-Loop and Approval Boundaries

    Place a human gate only where it earns its cost, pick the right interrupt (approve-before, review-after, or escalate-on-trigger), write explicit escalation criteria, and say what feedback changes — memory, policy, or weights.

  15. Lesson 0015

    The Design Spine: An End-to-End Agent Sketch

    One repeatable template that turns "design an agent" into eight ordered passes — frame, fit, loop, tools, memory, gates, monitoring, failure modes — and produce one full architecture sketch with it.

  16. Lesson 0016

    Claude Code Configuration and CI/CD Workflows

    Place a convention at the right Claude Code surface and sharing level, choose plan mode vs direct execution by scope, and wire a CI review so its findings are trustworthy.
