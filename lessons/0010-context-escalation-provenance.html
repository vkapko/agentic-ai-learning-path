Lesson 0010

Preserve The Case File

Tangible win: design a context packet that keeps exact facts, escalation reasons, and claim-source links intact as an agent workflow gets longer.

The One Idea

Context management is not just token budgeting. It is reliability design. The Claude architecture notes warn that progressive summaries can blur numbers, dates, and customer-stated expectations, and that downstream agents need structured metadata rather than loose prose [Claude context notes]. The long-context paper Lost in the Middle found that model performance can degrade when relevant information appears in the middle of long inputs.

Keep the compact version open: Context, Escalation, and Provenance Reference.

Minimum Rule

Store exact case facts outside the conversation summary, propagate structured errors instead of vague failures, escalate on explicit criteria, and preserve claim-source mappings through every synthesis step.

Four Durable Artifacts

Facts

Case facts

Persist IDs, amounts, dates, statuses, user requests, policy results, and unresolved assumptions in a compact block.

Errors

Error packets

Pass failure type, attempted action, retryability, partial results, and alternatives to the coordinator.

Escalate

Escalation rules

Escalate on human request, policy gap, authority limit, contradiction, safety issue, or no meaningful progress.

Sources

Claim-source map

Attach each important claim to source ID, excerpt or location, publication date, and confidence or conflict status.

Context Is Not Memory

Conversation history is useful for flow, but it is a weak home for facts that must survive compression. Put durable facts in a controller-owned case file, trim verbose tool outputs before they accumulate, and place the current case summary near the beginning of the prompt so the model does not have to recover critical facts from the middle of a long transcript [context preservation notes].

Choose The Preservation Move

Pick the move that keeps control information usable after the workflow gets long.

Design Drill: Billing Dispute Case Facts

A support agent handles a billing dispute across several turns. Draft the persistent case facts block that should survive summarization.

{
  "customerRequest": "dispute duplicate charge and wants refund",
  "identifiers": { "customerId": "cust_1842", "invoiceId": "inv_771" },
  "amountsAndDates": [
    { "amount": "49.00", "currency": "USD", "date": "2026-06-14", "source": "billing_api" }
  ],
  "policyStatus": "refund allowed if duplicate charge verified",
  "evidenceSources": ["billing_api:charge_1", "crm:case_928"],
  "openQuestions": ["second charge ID not yet confirmed"],
  "escalationStatus": "not escalated; escalate if duplicate cannot be verified"
}

Design Drill: Human Handoff Packet

Write a handoff packet for a price-match request blocked by an ambiguous policy gap.

Escalation trigger: policy gap for competitor price match after shipment. Case facts: order ord_123 shipped on 2026-06-18, customer requests $22 adjustment, customer provided competitor listing. What was attempted: checked order, checked standard price-adjustment policy, verified listing URL was reachable. Partial evidence: policy covers own-site price drops but not competitor listings. Decision needed: whether to approve exception or deny with explanation. Customer-facing status: "I need a specialist to review this policy exception."

Design Drill: Claim-Source Map

Convert this loose synthesis note into a structured claim-source record: "Refunds increased after the policy change, but the support team says the exact effect is unclear."

{
  "claim": "Refund volume increased after the policy change, but causal effect is unclear.",
  "sources": [
    {
      "sourceId": "dashboard:refunds_weekly",
      "dateRange": "2026-05-01/2026-06-15",
      "locationOrExcerpt": "weekly refund count rose from 420 to 515"
    },
    {
      "sourceId": "interview:support_lead",
      "date": "2026-06-16",
      "locationOrExcerpt": "support lead said seasonality may explain part of the rise"
    }
  ],
  "uncertainty": "correlation observed; causality not established",
  "conflictStatus": "not contradictory; needs method check"
}

Architecture Check

  • Which exact values must never be compressed into vague prose?
  • Which tool outputs should be trimmed before entering model context?
  • Which errors need structured packets instead of generic status text?
  • Which escalation triggers are explicit and testable?
  • Which claims require source IDs, dates, excerpts, or conflict notes?
  • Where will the controller store the case file between turns and agents?
