# Agentic AI Architecture Resources

## Knowledge

- [Local: Agentic AI Architecture Foundations](./agentic-ai-architecture-foundations-designing-autonomous-ai-systems.md)
  Use for: core vocabulary, agent types, perception/reasoning/action/learning modules, monitoring, and centralized/decentralized/hybrid architectures.
- [Local: Agentic AI Solution Design Patterns](./agentic-ai-solution-design-patterns.md)
  Use for: ReAct, chain of thought, reflection, Reflexion, tool discovery, human-in-the-loop, multi-agent patterns, and shared memory.
- [Local: Agentic AI Planning and Reasoning Design Patterns](./agentic-ai-planning-and-reasoning-design.md)
  Use for: plan-and-execute, concurrent execution, ReWOO, planner-critic-refiner, LATS, Self-Discover, verifier agents, memory, in-context learning, and adaptive tool orchestration.
- [Book: Artificial Intelligence: A Modern Approach](https://aima.cs.berkeley.edu/)
  Use for: classical agent definitions, agent-environment loops, reflex/model-based/goal-based/utility-based agents, and search/planning foundations.
- [Paper: Intelligent Agents: Theory and Practice](https://www.cs.ox.ac.uk/people/michael.wooldridge/pubs/ker95.pdf)
  Use for: autonomy, reactivity, proactiveness, and social ability as core agent properties.
- [Paper: ReAct: Synergizing Reasoning and Acting in Language Models](https://arxiv.org/abs/2210.03629)
  Use for: reason-act-observe loops and tool/environment interaction patterns.
- [NIST AI Risk Management Framework 1.0](https://nvlpubs.nist.gov/nistpubs/ai/NIST.AI.100-1.pdf)
  Use for: safety, validity, reliability, privacy, transparency, explainability, fairness, and governance framing.
- [Paper: Chain-of-Thought Prompting Elicits Reasoning in Large Language Models](https://arxiv.org/abs/2201.11903)
  Use for: when decomposition improves reasoning performance and when extra reasoning cost may be justified.
- [Paper: Language Models Don't Always Say What They Think](https://arxiv.org/abs/2305.04388)
  Use for: limits of visible reasoning and why reasoning traces are not proof of correctness.
- [Paper: Self-Refine: Iterative Refinement with Self-Feedback](https://arxiv.org/abs/2303.17651)
  Use for: reflection loops, draft improvement, and self-feedback limits.
- [Paper: Reflexion: Language Agents with Verbal Reinforcement Learning](https://arxiv.org/abs/2303.11366)
  Use for: memory-based episode learning without changing model weights.
- [Paper: An LLM Compiler for Parallel Function Calling](https://arxiv.org/abs/2312.04511)
  Use for: converting plans into parallelizable task graphs and understanding concurrency tradeoffs.
- [Paper: ReWOO: Decoupling Reasoning from Observations](https://arxiv.org/abs/2305.18323)
  Use for: information-heavy tasks where retrieval can be planned up front.
- [Paper: Language Agent Tree Search](https://arxiv.org/abs/2310.04406)
  Use for: search-based agent reasoning and exploring multiple action paths before commitment.
- [Paper: Self-Discover](https://arxiv.org/abs/2402.03620)
  Use for: generated reasoning structures for novel tasks.
- [Paper: Let's Verify Step by Step](https://arxiv.org/abs/2305.20050)
  Use for: verifier-style second-pass checking and independent evaluation.
- [Paper: Generative Agents](https://arxiv.org/abs/2304.03442)
  Use for: memory, reflection, and simulated multi-agent behavior.
- [Paper: Voyager](https://arxiv.org/abs/2305.16291)
  Use for: procedural memory and open-ended skill acquisition in embodied/digital environments.
- [Survey: In-Context Learning](https://arxiv.org/abs/2301.00234)
  Use for: retrieved examples, memories, and prompt context without model-weight updates.
- [LangGraph documentation: Workflows and agents](https://docs.langchain.com/oss/python/langgraph/workflows-agents)
  Use for: practical workflow/agent distinctions and implementation framing.
